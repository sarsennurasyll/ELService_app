# ELService: обзор проекта

## Назначение

ELService — мобильное приложение и backend API для маркетплейса услуг по ремонту бытовой техники.

Проект состоит из нескольких частей:

- Flutter-приложение в корне проекта.
- Backend API в папке `backend`.
- React/Vite дизайн-референс в папке `ELService-design`.

React/Vite проект используется как эталон UI. Flutter-приложение переносит дизайн из него с использованием собственной дизайн-системы на Flutter.

## Цель

Цель проекта — дать клиенту простой способ создать заявку на ремонт бытовой техники, получить предложения от мастеров, выбрать исполнителя, общаться с ним в чате и завершить заказ.

Для мастера проект должен дать возможность видеть заказы, отправлять предложения и вести работу по принятым заказам.

Для администратора проект должен дать базовый каркас управления заказами, пользователями, аналитикой и настройками.

## Бизнес-идея

ELService работает как marketplace:

1. Клиент создаёт заказ на ремонт.
2. Мастера видят подходящий заказ.
3. Мастер отправляет предложение с ценой, временем приезда и комментарием.
4. Клиент принимает одно предложение.
5. Заказ получает назначенного мастера.
6. Для клиента и мастера создаётся чат.
7. Мастер выполняет ремонт.
8. Клиент оплачивает услугу и оставляет отзыв.

Часть этого сценария уже реализована, часть пока не реализована.

## Роли пользователей

### Customer

Клиент создаёт заказы, смотрит категории услуг, просматривает свои заказы, принимает предложения мастеров, открывает профиль и может редактировать данные профиля.

Реализовано частично:

- регистрация и вход;
- просмотр категорий;
- создание заказа;
- просмотр списка заказов;
- просмотр деталей заказа;
- просмотр профиля;
- редактирование профиля;
- просмотр предложений по заказу;
- принятие предложения;
- backend-чат и Flutter data-слой чата.

Не реализовано полностью:

- оплата;
- отзыв после ремонта;
- реальные уведомления;
- полноценное подключение UI списка чатов к backend;
- полноценный маршрут на экран сообщений.

### Technician

Мастер должен получать заказы, отправлять предложения, вести календарь, видеть заработок и профиль.

Реализовано частично:

- Flutter-страницы-заготовки для dashboard, orders, calendar, earnings, profile;
- нижняя навигация мастера;
- backend endpoint отправки предложения мастером;
- Flutter экран отправки предложения.

Не реализовано полностью:

- авторизационное разделение интерфейсов по ролям;
- получение персонального списка заказов мастера;
- wallet/earnings через backend;
- календарь через backend.

### Admin

Администратор должен управлять заказами, пользователями, аналитикой и настройками.

Реализовано частично:

- Flutter-страницы-заготовки для dashboard, orders, users, analytics, settings;
- нижняя навигация администратора;
- backend роль `ADMIN` используется в проверках доступа для Offers.

Не реализовано:

- backend endpoints администрирования;
- управление пользователями;
- аналитика;
- отчёты;
- платежи;
- модерация.

## Основные возможности проекта

### Реализовано

- Flutter-проект с Material 3.
- Feature First + Clean Architecture.
- Базовая дизайн-система Flutter.
- UI Kit: кнопки, поля ввода, карточки, верхняя панель, экранная обёртка, нижняя навигация.
- GoRouter с `StatefulShellRoute.indexedStack` для сохранения состояния вкладок.
- Node.js/Express backend.
- PostgreSQL через Prisma.
- JWT-авторизация.
- Регистрация, вход, refresh, logout.
- Профиль пользователя.
- Категории.
- Заказы.
- Предложения мастеров.
- Создание чата после принятия предложения.
- HTTP endpoints для списка чатов, сообщений и отправки сообщения.
- Flutter data/domain слой для Auth, Categories, Orders, User Profile, Offers, Chat.
- Локальная конфигурация Flutter API base URL: `http://localhost:3000/api/v1`.
- CORS для локальной разработки Flutter Web на случайных `localhost` портах.

### Не реализовано

- Firebase.
- WebSocket.
- Push-уведомления.
- Оплата.
- Отзывы через API.
- Полноценное разделение стартового сценария по ролям.
- Темная тема.
- Локализация.
- Sentry/Crashlytics.
- Автоматические тесты бизнес-сценариев.
- Seed-данные для категорий.
- Полная интеграция Chat UI с backend.
